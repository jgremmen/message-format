# ASM

The `message-format-asm` module provides `AsmAnnotationAdopter`, a concrete implementation of
[`AbstractAnnotationAdopter`](../annotations/adopter.md) that uses the
[ASM bytecode library](https://asm.ow2.io/) to scan compiled class files for
[`@MessageDef` and `@TemplateDef`](../annotations/index.md) annotations. Because ASM reads the
bytecode directly, the scanned classes never need to be loaded into the JVM. This makes the
adopter suitable for build-time processing, plugin environments and any situation where class
loading is undesirable or unavailable.


## Usage

Create an `AsmAnnotationAdopter` by passing your `ConfigurableMessageSupport` instance. Then use
any of the discovery strategies inherited from `AbstractAnnotationAdopter` to adopt annotated
classes:

```java
var messageSupport = MessageSupportFactory.create(
    DefaultFormatterService.getSharedInstance());

var adopter = new AsmAnnotationAdopter(messageSupport);

// Scan entire packages on the classpath
adopter.adopt(
    Thread.currentThread().getContextClassLoader(),
    Set.of("com.example.messages"));
```

Individual classes can be adopted by type reference, file path, or `Path`:

```java
adopter.adopt(MyMessages.class);
adopter.adopt(Path.of("target/classes/com/example/Templates.class"));
```

Because all `adopt` methods return the adopter itself, multiple calls can be chained:

```java
adopter
    .adopt(AuthMessages.class)
    .adopt(OrderMessages.class)
    .adopt(CommonTemplates.class);
```

If the factory and publisher need to be provided independently, a second constructor accepts a
`MessageFactory` and a `MessagePublisher`:

```java
var adopter = new AsmAnnotationAdopter(messageFactory, publisher);
```


## How It Works

When a class file is presented to the adopter, it is passed to an ASM `ClassReader` with the
flags `SKIP_CODE`, `SKIP_DEBUG` and `SKIP_FRAMES`. This tells ASM to skip method bodies, debug
information and stack map frames entirely, which keeps the scan fast and memory-efficient since
only the annotation metadata is of interest.

The adopter visits annotations on the class itself and on every non-synthetic method. Synthetic
methods, which are generated by the compiler for features like bridge methods or lambda
expressions, are ignored because they are not meaningful targets for message annotations.
Annotations are recognized in both their singular form (`@MessageDef`, `@TemplateDef`) and their
repeatable container form (`@MessageDefs`, `@TemplateDefs`).

The highest ASM API version available on the classpath is detected automatically at class loading
time. This means the adopter works with any ASM release in the supported range without requiring
configuration changes.


## Complete Example

The following example sets up a `MessageSupport`, scans a package for annotated classes, and
formats the resulting messages:

```java
// Set up message support and adopter
var messageSupport = MessageSupportFactory.create(
    DefaultFormatterService.getSharedInstance());
var adopter = new AsmAnnotationAdopter(messageSupport);

// Scan for annotated classes
adopter.adopt(
    Thread.currentThread().getContextClassLoader(),
    Set.of("com.example"));

// Format a message defined via @MessageDef(code = "welcome", ...)
String greeting = messageSupport
    .code("welcome")
    .with("name", "Alice")
    .locale(Locale.ENGLISH)
    .format();
```


## Module Coordinates

```
de.sayayi.lib:message-format-asm:<version>
```

This module depends on `message-format-annotations` (which transitively includes the
`message-format` core) and on the ASM library `org.ow2.asm:asm` (version 9.x). When using
Gradle:

```groovy
dependencies {
    implementation 'de.sayayi.lib:message-format-asm:<version>'
}
```

For Maven:

```xml
<dependency>
    <groupId>de.sayayi.lib</groupId>
    <artifactId>message-format-asm</artifactId>
    <version><!-- version --></version>
</dependency>
```

If your project already uses the [Spring module](../spring/index.md), you do not need this module
separately. Spring bundles its own copy of ASM, and the `SpringAsmAnnotationAdopter` provided by
that module offers the same scanning functionality without adding an extra ASM dependency.
